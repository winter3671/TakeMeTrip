# Conventional Commits (컨벤셔널 커밋) 추가 규칙
더욱 구조화된 커밋 메시지를 위해 타입(Type)을 앞에 붙입니다. 
- feat: 새로운 기능 추가 (Feature).
- fix: 버그 수정 (Bug Fix).
- docs: 문서 관련 변경 (Documentation).
- style: 코드 스타일, 포맷 등 코드 자체 변경 없는 수정 (Formatting, Changes to code format).
- refactor: 코드 리팩토링 (코드 기능 변경 없이 구조 개선).
- test: 테스트 코드 관련 변경.
- chore: 빌드 프로세스, 의존성 등 기타 (Build, CI).
